## Замечания по реализации

- Использован HTML вместо MarkdownV2 (визуально результат тот же, HTML
  устойчивее из-за меньшего числа экранируемых символов).
- file_id-кэш для баннеров не реализован — каждый раз бот заливает PNG
  в Telegram заново. Для 100 учеников приемлемо (~100 МБ трафика/мес
  при активном использовании), для большего объёма стоит добавить кэш.
- Один и тот же `trial` баннер показывается на всех 5 шагах booking-FSM.
  На последних шагах визуально не идеально, но edit_photo внутри aiogram
  не поддерживается без отправки нового сообщения — оставили как есть.
