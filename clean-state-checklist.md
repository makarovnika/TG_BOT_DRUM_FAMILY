# Clean State Checklist

Пройди этот список **перед окончанием каждой сессии**. Если хоть один пункт
не отмечен — допиши недостающее или явно зафиксируй причину в
`claude-progress.md` и `session-handoff.md`.

- [ ] Стандартный путь старта (`./init.sh`) всё ещё работает.
- [ ] Стандартный путь верификации (`uv run pytest -q`, `uv run ruff check`) всё ещё запускается.
- [ ] Текущий прогресс зафиксирован в `claude-progress.md` (новая запись `### Session NNN`).
- [ ] `feature_list.json` отражает реальность: что прошло верификацию (`passing`),
      что в работе (`in_progress`), что заблокировано (`blocked`).
- [ ] Ни один полусделанный шаг не остался без записи в `session-handoff.md`.
- [ ] Следующая сессия сможет продолжить без ручного ремонта (нет «висящих»
      миграций, незакоммиченных конфигов, запущенных процессов в фоне).
- [ ] Секреты (BOT_TOKEN, YClients-токены) не попали в коммит. `.env` в `.gitignore`.
- [ ] Если фича переведена в `passing` — её `evidence` содержит конкретные
      ссылки/выводы команд/скриншоты, а не пустой массив.

---

## Зачем это нужно

Чек-лист защищает от типичной ловушки: «вроде работает, но следующий запуск
падает, и неясно почему». Шесть пунктов покрывают старт, верификацию, журнал,
честность статусов, документирование «полусделанного» и готовность к
продолжению — это всё, что нужно, чтобы между сессиями не накапливался
скрытый технический долг.
