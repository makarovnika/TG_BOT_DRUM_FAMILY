# AGENTS.md

Этот репозиторий рассчитан на длительную работу coding-агента. Цель — не
максимум сгенерированного кода, а оставить репозиторий в таком состоянии,
чтобы следующая сессия могла продолжить работу без догадок.

## Startup Workflow

Перед тем как писать код:

1. Подтверди рабочую директорию: `pwd`.
2. Прочитай `claude-progress.md` — последнее подтверждённое состояние и следующий шаг.
3. Прочитай `feature_list.json` и выбери фичу с наивысшим приоритетом из незавершённых.
4. Посмотри последние коммиты: `git log --oneline -5`.
5. Запусти `./init.sh`.
6. Прогони обязательную smoke- или end-to-end-проверку до начала новой работы.

Если базовая верификация уже падает — чини её первой. Не стакай новую работу
поверх сломанного старта.

## Working Rules

- Работаем над одной фичей за раз.
- Не помечаем фичу «готовой» просто потому, что добавлен код.
- Изменения держим в рамках выбранной фичи, кроме узкого блокирующего фикса.
- Не меняем правила верификации молча по ходу реализации.
- Долговечные артефакты репозитория важнее саммари в чате.

## Required Artifacts

- `feature_list.json` — источник правды по статусу фич
- `claude-progress.md` — журнал сессий и текущее подтверждённое состояние
- `init.sh` — стандартный путь старта и верификации
- `session-handoff.md` — компактный handoff для крупных сессий

## Definition Of Done

Фича считается завершённой, только если выполнено всё перечисленное:

- целевое поведение реализовано
- требуемая верификация **реально запускалась**
- доказательства зафиксированы в `feature_list.json` или `claude-progress.md`
- репозиторий остаётся перезапускаемым через стандартный startup path

## End Of Session

Перед окончанием сессии:

1. Обнови `claude-progress.md`.
2. Обнови `feature_list.json`.
3. Зафиксируй нерешённые риски или блокеры.
4. Закоммить с осмысленным сообщением, когда работа в безопасном состоянии.
5. Оставь репозиторий достаточно чистым, чтобы следующая сессия сразу могла
   запустить `./init.sh`.
